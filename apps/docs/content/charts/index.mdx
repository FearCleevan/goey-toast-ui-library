# @gooey/charts

Five themed chart components for React Native, built on [Victory Native](https://commerce.nearform.com/open-source/victory-native/). All charts inherit your design system colors automatically — zero extra config.

## Install

<Tabs items={["Expo", "pnpm", "npm", "yarn"]}>
  <Tabs.Tab>
  ```bash
  npx expo install @gooey/charts victory-native react-native-svg
  ```
  </Tabs.Tab>
  <Tabs.Tab>
  ```bash
  pnpm add @gooey/charts victory-native react-native-svg
  ```
  </Tabs.Tab>
  <Tabs.Tab>
  ```bash
  npm install @gooey/charts victory-native react-native-svg
  ```
  </Tabs.Tab>
  <Tabs.Tab>
  ```bash
  yarn add @gooey/charts victory-native react-native-svg
  ```
  </Tabs.Tab>
</Tabs>

<Callout type="info">
  `@gooey/charts` uses the **SVG-based** Victory Native API (`victory-native ≥ 36`). It does **not** require Skia or the Victory Native XL API.
</Callout>

## Theming

Charts auto-read colors from `@gooey/core`'s theme. No extra configuration is needed.

```tsx
import { BarChart } from "@gooey/charts";

// Uses your active theme colors automatically
<BarChart data={data} height={240} />

// Or override a specific color
<BarChart data={data} height={240} color="#7C3AED" />
```

## Toast integration

Every chart accepts a `toastConfig` prop that fires a `gooeyToast` when the user taps a data point:

```tsx
<BarChart
  data={salesData}
  height={240}
  toastConfig={{
    titleKey: "month",
    descriptionKey: "revenue",
    prefix: "$",
    duration: 3000,
  }}
/>
```

See [ChartTooltip](/charts/chart-tooltip) for the full config reference.
