# @gooey/cli

A shadcn-style component installer for gooey-toast. Copy components directly into your project — own the code, modify freely, no black-box abstraction.

## How to run

<Tabs items={["npx (no install)", "Global install"]}>
  <Tabs.Tab>
  ```bash
  npx @gooey/cli init
  npx @gooey/cli add Button
  ```
  </Tabs.Tab>
  <Tabs.Tab>
  ```bash
  npm install -g @gooey/cli

  gooey init
  gooey add Button
  ```
  </Tabs.Tab>
</Tabs>

## Quick start

<Steps>

### Initialize your project

```bash
npx @gooey/cli init
```

This detects your Expo SDK version and package manager, installs `@gooey/core`, and creates `gooey.config.json`.

### Browse available components

```bash
gooey list
```

### Add a component

```bash
gooey add SignInBlock
```

The component source file is copied to `src/components/gooey/SignInBlock.tsx`. Peer dependencies are installed automatically.

### Customize it

Open the file and edit it freely. You own it — the CLI won't overwrite your changes.

</Steps>

## How it works

`@gooey/cli` maintains a **registry** of component templates. When you run `gooey add`, it:

1. Looks up the component template in the registry
2. Copies the source file into `{outputDir}/{ComponentName}.tsx`
3. Installs any required peer dependencies using your detected package manager
4. Records the component in `gooey.config.json` so `gooey diff` can track drift

<Callout type="info">
  Unlike traditional npm packages, CLI-added components live **in your codebase**. You own the file and can change anything without waiting for a library update.
</Callout>

## gooey.config.json

Created by `gooey init` at your project root:

```json
{
  "outputDir": "src/components/gooey",
  "packageManager": "pnpm",
  "expo": true,
  "components": ["SignInBlock", "ProductCard"]
}
```

| Field | Description |
|---|---|
| `outputDir` | Directory where component files are copied |
| `packageManager` | Auto-detected: `pnpm`, `npm`, `yarn`, or `bun` |
| `expo` | Whether the project uses Expo (affects install commands) |
| `components` | List of components added to track drift with `gooey diff` |
