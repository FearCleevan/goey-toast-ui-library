# @gooey/cli

A shadcn-style component installer for goey-toast. Copy components directly into your project — own the code, customize freely.

## Install

```bash
npm install -g @gooey/cli
# or use npx without installing
npx @gooey/cli init
```

## Commands

| Command | Description |
|---|---|
| `gooey init` | Initialize a project — detects Expo/React Native, installs `@gooey/core`, creates `gooey.config.json` |
| `gooey add <component>` | Copy a component into your project and install its peer deps |
| `gooey list` | Show all available components with their tier and status |
| `gooey diff <component>` | Diff your local copy against the current registry version |

## How it works

`@gooey/cli` maintains a **registry** of component templates. When you run `gooey add`, it:

1. Copies the component source file into your configured output directory
2. Installs any required peer dependencies via your detected package manager
3. Records the component in `gooey.config.json` so `gooey diff` can track changes

<Callout type="info">
  Unlike traditional packages, CLI-added components live **in your codebase**. You own the file and can modify it freely.
</Callout>

## gooey.config.json

Created by `gooey init` at the project root:

```json
{
  "outputDir": "src/components/gooey",
  "packageManager": "pnpm",
  "expo": true,
  "components": ["SignInBlock", "ProductCard"]
}
```
